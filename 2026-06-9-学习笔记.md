# 网络与大模型基础学习笔记

## 一、大模型工具调用机制
- 大模型本身仅支持对话交互，调用外部工具需通过标准化接口
- **MCP 类比理解**：相当于“统一的USB接口”
  - 外部工具（API、数据库等）封装为 `MCP Server`（统一包装）
  - AI 作为 `MCP Client`，只要支持标准协议，即可调用所有兼容工具
  - 调用前提：基础权限、密钥、网络连通性

---

## 二、核心网络概念
API / URL / URI / TCP（通道） / 协议 / DNS / 域名 / 抓包 / 服务器（如淘宝等公司服务器，分工不同） / 操作系统 / 浏览器 / 客户端 / 服务端 / 万维网 / 单片机

---

## 三、完整上网流程
1.  浏览器先查本地缓存
2.  操作系统查缓存
3.  DNS 域名解析（把网址翻译成IP地址）
4.  建立 TCP 连接（和服务器建立通信通道）
5.  发送 HTTP/HTTPS 请求（告诉服务器你要什么）
6.  服务器处理请求并返回数据
7.  浏览器接收数据包，断开连接
8.  浏览器渲染画面（把数据变成你看到的网页）

---

## 四、浏览器的核心作用
- 接受用户指令
- 找服务器地址（域名解析）
- 连服务器、发请求
- 收数据、翻译并展示画面

---

## 五、网络通信的本质
网络是由很多计算机等设备相互连接组成的，通信过程中必须：
1.  确定正确的通信对象（IP地址+端口）
2.  把请求和响应准确发送给目标对象
