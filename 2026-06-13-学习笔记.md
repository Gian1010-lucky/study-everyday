计算机网络与安全知识点汇总
 
文档说明
 
本文档整理所有网络、系统、安全相关的提问与知识点，按类别划分，采用通俗解释+案例的形式，方便复习查阅。
 
 
 
一、网络基础核心概念
 
1. IP地址
 
- 定义：互联网协议地址，是设备在网络中的「门牌号」，用来定位设备、实现跨网络数据传输。
- 分类：分为内网IP（局域网内使用）和公网IP（互联网唯一地址）。
- 关键认知：IP地址不是电脑天生自带的，是联网后由网络分配的——内网IP由路由器分配，公网IP由运营商分配。断开网络会释放IP，重新连接可能变化，也可手动设置为固定IP。
 
2. MAC地址
 
- 定义：媒体访问控制地址，是网卡的「物理身份证号」，出厂时烧录在网卡硬件中，全球唯一、永久不变。
- 作用：局域网内设备通信靠MAC地址寻址，相当于同个小区内送快递靠房间号+收件人。
- 与IP地址的关系：
- IP是「可变门牌号」，换一个网络就会变更；MAC是「固定身份证」，永远跟随网卡。
- 同局域网内通信：优先用MAC地址直接传输，无需IP路由。
- 跨网/访问外网：靠IP地址寻址规划路线，到达目标局域网后，再通过MAC地址送到具体设备。
 
3. 内网（局域网）与公网
 
内网 / 局域网（LAN）
 
- 定义：小范围的本地网络，比如家庭、宿舍、公司、机房的内部网络，设备之间可直接互通，不直接接入互联网。
- 特点：
- 使用专属内网IP段（常见： 192.168.x.x 、 10.x.x.x ），仅在内部生效，全球重复也不冲突。
- 传输速度快、延迟低，自带天然隔离性，外网无法直接主动访问内网设备。
- 核心设备：交换机负责内网数据转发，路由器负责对接外网。
- 通信逻辑：
- 内网设备互传：仅走交换机，通过MAC地址直达，不经过外网，速度极快。
- 内网访问外网：必须经过网关（路由器）做地址转换，才能接入互联网。
 
公网（互联网）
 
- 定义：全球连通的公共网络，由无数路由器、服务器组成。
- 特点：公网IP全球唯一，能被互联网上所有设备直接定位。
 
4. 子网与子网掩码
 
- 子网：将一个大的局域网，切分成多个独立的小型网段，每个小网段就是一个子网，作用是隔离流量、提升安全性、方便管理。
- 子网掩码：用来判断两个IP是否处于同一子网的「判定标尺」。
- 常见格式： 255.255.255.0 
- 通俗理解：它和IP地址做运算后可得到网段号，网段相同的设备可直接通信，网段不同则必须通过网关转发。
 
5. 网关
 
- 定义：一个网络的「出口大门」，家庭网络中通常就是路由器（常见地址如 192.168.1.1 ）。
- 作用：
- 内网设备访问外网，必须先把数据发给网关，由网关转发到互联网。
- 外网回传的数据，也先到达网关，再由网关转发给内网对应的设备。
- 不同子网之间通信，也需要网关做中转转发。
 
6. 广播地址
 
- 定义：向一个网段内所有设备同时发送消息的特殊地址。
- 分类：
1. 全局广播地址（ 255.255.255.255 ）：向所有网络发送广播，路由器默认会拦截全局广播，禁止其扩散到外网。
2. 定向广播地址：向指定子网内所有设备发广播，比如 192.168.1.255 就是 192.168.1.0 网段的定向广播地址。
- 全局与定向广播隔离：路由器默认不转发全局广播，也会限制跨子网的定向广播，防止广播风暴、避免攻击大范围扩散。
 
7. DHCP
 
- 定义：动态主机配置协议，通俗说就是自动给设备分配IP地址的服务，一般由路由器提供。
- 作用：设备联网时，无需手动填写IP、子网掩码、网关，DHCP会自动分配好参数，避免IP冲突，实现即插即用。
- 为什么需要自动分配：手动设置IP容易和其他设备冲突，且更换网络就要重新配置；DHCP自动适配所有网络，使用更便捷。
 
8. IDC（互联网数据中心）
 
- 定义：互联网数据中心，本质是专业级的大型服务器机房，是互联网的核心基础设施。
- 内部构成：集中放置服务器、存储设备、交换机，配套不间断供电、恒温冷却、安防监控系统。
- 核心作用：为企业提供服务器托管、带宽、云服务等资源；网站、APP、游戏的服务器大多部署在IDC机房中，保障24小时稳定运行。
 
 
 
二、特殊网络与隔离机制
 
1. 光明网与物理隔离
 
- 光明网是什么：我国政务专用内网，属于高安全等级的专用局域网，全程与互联网物理断开。
- 物理隔离的原理：
并非依靠软件、防火墙限制，而是物理线路完全不连通——内网使用独立的网线、光纤、路由器和服务器，与外网（互联网）没有任何物理线路连接。
相当于两个完全独立的房间，没有门也没有通道，数据完全无法跨网络传输。
- 用途：政务、涉密、科研、高校内部系统，防止黑客入侵与数据泄露。
 
2. 为什么内网专属软件外部无法使用
 
以朝鲜内网软件、政务内网系统为例，核心原因有三点：
 
1. 服务器物理隔离：软件的后台服务器仅部署在内网中，与互联网无物理连接，外网设备无法寻址、无法连接。
2. 网络环境不互通：普通手机流量、家庭宽带走的都是公网，无法接入物理隔离的内网；必须使用内网专线、内网专属设备才能访问。
3. 权限与白名单限制：即使网络连通，内网系统通常也有严格的设备白名单、身份认证，外部设备无权限接入。
 
3. 内网为什么外部设备无法主动访问
 
1. 地址不可达：内网IP仅在局域网内部生效，公网没有内网IP的路由路径，外网设备无法定位到内网设备。
2. NAT地址转换：路由器做了网络地址转换，内网所有设备对外共用一个公网IP，外网无法直接定位到具体某台内网电脑。
3. 防火墙默认拦截：路由器、防火墙默认拦截所有外网主动向内网发起的连接请求，仅放行内网主动向外请求的回应数据。
4. 高安全内网物理断连：政务、涉密内网直接物理隔离，完全没有数据通路。
 
4. 私有云
 
- 定义：专属云服务，仅面向特定个人或企业开放，部署在自有机房或专属服务器中，不对外公开。
- 特点：数据自主掌控、安全性高、支持深度定制，适合企业内部办公、涉密数据存储场景。
- 与公有云区别：公有云（阿里云、百度云）面向所有用户开放售卖；私有云为专属自用，不对外开放。
 
5. NAS
 
- 定义：网络附加存储，通俗说就是连网的私人硬盘/私有网盘。
- 作用：接入家庭路由器后，内网所有设备都可访问、存取文件、播放视频，相当于本地专属云盘，数据存在本地，无需充值会员。
- 适用场景：家庭存储照片视频、小型公司内部文件共享。
 
 
 
三、常用网络命令与工具
 
1. curl命令
 
核心定位
 
命令行网络工具，核心作用是在终端发起网络请求、收发网络数据，相当于「没有图形界面的浏览器」。
 
主要功能
 
1. 访问网页/接口，查看返回内容
2. 测试API接口，发送GET/POST请求、传递参数
3. 下载/上传文件
4. 查看网站响应头、检测网站运行状态
5. 支持HTTP/HTTPS、FTP等多种网络协议
 
核心原则（必记）
 
curl在哪台设备上运行，哪台设备就是主动发起请求的一方。它无法跨设备强行向另一台电脑推送文件，只能控制当前设备主动执行下载/上传操作。
 
黑客攻击场景原理
 
黑客无法用自己电脑的curl直接向你发送文件（会被防火墙拦截），真实攻击流程为：
 
1. 黑客先通过漏洞入侵，获取你电脑的操作权限
2. 远程操控你的电脑运行curl命令
3. 让你的电脑主动访问黑客服务器，下载恶意文件，或主动把本地文件上传给黑客
 
学习周期
 
- 入门（基础用法）：半天即可掌握
- 常用进阶（接口测试、高频参数）：1~3天
- 全量参数：上百个，无需全部背诵，用到时查阅文档即可。
 
2. ping命令
 
- 定义：最基础的网络连通性测试工具，基于ICMP协议实现。
- 原理：向目标地址发送小型探测数据包，目标收到后返回应答包，通过往返时间计算网络延迟，无应答则表示网络不通。
- 常用场景：检测网络是否断开、测试网站能否访问、排查路由器连通性。
 
 
 
四、网络安全基础
 
1. XSS攻击（含PDF-XSS）
 
- 定义：跨站脚本攻击，黑客在网页或文件中植入恶意脚本代码，用户打开后，脚本会在用户浏览器中静默执行。
- PDF-XSS：XSS的变种，在PDF文件中植入恶意脚本，用户打开带毒PDF时脚本触发执行。
- 常见危害：窃取用户Cookie、冒充用户登录账号、跳转至钓鱼网站。
 
2. Cookie窃取
 
- Cookie是什么：网站存储在用户浏览器中的「身份凭证」，记录登录状态、偏好设置，避免每次访问都重新登录。
- Cookie窃取：黑客通过XSS等手段，盗取用户浏览器中的Cookie，凭借该凭证冒充用户身份登录账号，无需知道账号密码。
 
3. Smurf攻击
 
- 定义：典型的拒绝服务（DoS）攻击，利用广播机制和IP伪造实现流量放大，俗称「借刀杀人」式攻击。
- 原理：
1. 黑客伪造受害者的IP地址，向某个子网的广播地址发送大量ICMP（ping）请求
2. 该子网内所有设备收到请求后，都会向伪造的源IP（受害者）返回应答包
3. 流量被几十上百倍放大，淹没受害者的网络带宽，导致其无法正常联网
- 防御方式：路由器禁用广播转发、主机关闭广播响应。
 
4. DPI（深度包检测）
 
- 定义：深度报文检测，是高级网络流量检测技术，不局限于检测IP和端口，还会拆解数据包，解析应用层内容。
- 检测能力：可识别HTTP/HTTPS、FTP、SSH、DNS等应用协议，也能识别具体网站、文件内容、异常行为。
- 可检测的攻击：端口扫描、SQL注入、XSS、木马外联、异常流量、数据窃取行为等。
- 检测盲区：
1. 强加密流量（HTTPS、TLS 1.3、SSH）：内容加密后，DPI仅能看到连接信息，无法解析具体内容
2. 低速、分片、伪装流量：如慢连接、分片传输，DPI容易漏判
3. 零日攻击、无特征恶意软件：无匹配的特征库，无法识别
 
5. 攻击流量检测总结
 
- 明文流量、有已知特征的攻击：DPI基本都能精准检测。
- 内网横向移动、异常外联、数据窃取：可通过行为分析（流量大小、连接频率、访问时间）发现。
- 整体结论：DPI擅长检测已知威胁，对未知攻击、加密攻击、慢速攻击效果有限，需配合行为分析、AI建模提升检测率。
 
 
 
五、操作系统基础
 
1. Windows / macOS / Linux 三大系统核心区别
 
出品方
 
- Windows：微软出品
- macOS：苹果出品
- Linux：开源社区维护，免费使用
 
内核类型
 
- Windows：NT内核，设计偏向图形交互、兼容性、大众易用
- macOS：Darwin内核，基于Unix开发
- Linux：纯Unix家族内核
 
核心定位
 
- Windows：面向普通个人/办公，图形界面为主，家用、办公、游戏主流
- macOS：仅预装在苹果电脑，兼顾颜值、办公、开发
- Linux：分支极多（Ubuntu、CentOS等），主打服务器、开发、运维、嵌入式场景，个人桌面使用较少
 
开源属性
 
- Windows：闭源
- macOS：闭源
- Linux：完全开源
 
文件路径格式
 
- Windows：使用反斜杠，示例： C:\文件夹\文件 
- macOS：使用正斜杠，示例： /文件夹/文件 
- Linux：使用正斜杠，示例： /文件夹/文件 
- 阵营划分：macOS与Linux同属Unix体系，底层血缘更近；Windows为独立技术体系。
 
2. curl跨平台通用的原理
 
- curl是独立的第三方应用软件，并非系统内核自带的功能。
- 开发者分别为Windows、macOS、Linux制作了适配版本，三个版本底层分别适配各自的系统内核，但对外的命令语法、功能完全一致。
- 通俗类比：微信在安卓、苹果、Windows上都能使用，操作逻辑相同，但底层适配了不同系统，curl的跨平台原理与此一致。
- 补充说明：Linux/macOS系统默认自带curl；Windows 10/11新版PowerShell已内置，老版本需手动安装。
 
 
 
六、网络通信流程示例
 
场景1：同局域网内两台电脑传文件
 
1. 电脑A通过ARP协议，获取电脑B的MAC地址
2. 数据封装MAC地址，通过交换机直接发送到电脑B
3. 全程不经过路由器、不走外网，速度快、延迟低
 
场景2：电脑访问百度网站（跨网通信）
 
1. 电脑（ 192.168.1.10 ）识别到目标是外网IP，先将数据发送给网关（路由器 192.168.1.1 ）
2. 路由器执行NAT地址转换，将内网IP替换为公网IP，转发至互联网
3. 互联网中的路由器通过IP地址寻址，将数据送达百度服务器
4. 百度返回数据沿原路回到路由器，路由器再转换回内网IP
5. 最终通过MAC地址将数据送达你的电脑
