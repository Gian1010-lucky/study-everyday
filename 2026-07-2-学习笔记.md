## 一、什么是计算机网络？

### 1\.1 基本定义

**计算机网络**是将地理位置不同的、具有独立功能的多台计算机及其外部设备，通过通信线路连接起来，在网络操作系统、网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。

简单来说：**网络 = 节点 \+ 链路 \+ 协议**

### 1\.2 网络的目的

- 实现数据上传、下载

- 网页浏览

- 在线游戏

- 视频通话

- 资源共享（文件、打印机等）

- 分布式计算

---

## 二、网络的组成：节点与链路

### 2\.1 节点（Node）

网络中的节点分为两大类：

|节点类型|说明|举例|
|---|---|---|
|**主机节点（端系统）**|位于网络边缘，直接面向用户|电脑、手机、服务器、IoT设备|
|**数据交换节点**|位于网络核心，负责数据转发|路由器、交换机、负载均衡器|

> **端系统（End System）**：就是网络边缘的主机设备，它由三部分组成：
> 
> 1. 硬件设备本身（电脑、手机等）
> 
> 2. 操作系统（Windows、Linux、iOS等）
> 
> 3. 网络应用（浏览器、游戏、微信等）
> 
> 

### 2\.2 链路（Link）

**链路**是把节点连接在一起的通信线路。

⚠️ **注意**：链路不仅仅是物理的网线或光纤，还包括支持链路运转的**链路层协议**。

#### 链路的分类

|分类方式|类型|说明|
|---|---|---|
|按位置分|**接入链路**|连接主机与边缘交换节点的链路（通常是"方形"拓扑）|
||**骨干链路**|连接核心交换节点之间的链路（通常是"圆形/环形"拓扑）|
|按介质分|有线链路|光纤、双绞线、同轴电缆|
||无线链路|WiFi、5G、蓝牙|

> 📝 **你的笔记修正**：
> 
> - "bank部" → **骨干**（Backbone）
> 
> - "观链路" → **骨干链路**
> 
> - "和方形有关的是接入链路，圆形是骨干链路" → 这个说法指的是**网络拓扑图**的画法：接入网通常画成树状/方形结构，骨干网通常画成环形/网状结构。
> 
> 

#### 链路的重要指标：带宽

**带宽（Bandwidth）**是链路每秒能传输的数据量，单位是：

- bps（比特每秒）

- Kbps、Mbps、Gbps、Tbps

## 三、网络的三大组成部分

互联网可以从逻辑上分为三个部分：

```
┌─────────────────────────────────────────────────┐
│                   网络边缘                        │
│  （主机、服务器、手机等端系统 + 应用程序）          │
└────────────────────┬────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────┐
│                   网络接入                        │
│  （接入网：把端系统连接到网络核心）                 │
│  例如：家庭宽带、5G基站、校园网                    │
└────────────────────┬────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────┐
│                   网络核心                        │
│  （路由器、交换机等交换设备 + 骨干链路）            │
│  负责数据的路由和转发                              │
└─────────────────────────────────────────────────┘
```

### 3\.1 网络边缘（Network Edge）

- 位于网络的最外围

- 由端系统（主机）和运行在其上的应用程序组成

- 是用户直接接触的部分

### 3\.2 网络接入（Network Access）

- 也叫**接入网**

- 作用：把端系统连接到网络核心

- 常见的接入网技术：

    - 有线：以太网、光纤入户（FTTH）、ADSL

    - 无线：WiFi、5G/4G、卫星

### 3\.3 网络核心（Network Core）

- 由大量路由器和交换机互联而成

- 作用：为边缘主机提供数据转发服务

- 是互联网的"骨架"

## 四、网络分层模型 ⭐重点

### 4\.1 为什么要分层？

网络通信是一个非常复杂的过程，分层的好处：

1. **模块化设计**：每一层只负责自己的功能，独立设计和维护

2. **易于理解和学习**：把复杂问题分解成多个小问题

3. **互操作性好**：不同厂商的设备只要遵循同一层的协议就能互通

4. **便于升级**：某一层的技术升级不影响其他层

### 4\.2 分层的核心思想

> **每一层实现特定的功能，上一层借助下一层提供的服务，与对等层的协议实体交换协议报文。**
> 
> 

这句话是理解网络分层的关键，拆解一下：

- **对等层通信**：发送方的第N层和接收方的第N层"好像"在直接对话

- **实际传输**：数据是逐层向下传递，经过物理链路传到对方，再逐层向上

- **服务与被服务**：第N层为第N\+1层提供服务，第N\+1层使用第N层的服务

### 4\.3 OSI七层模型 ⭐⭐⭐重点

**OSI（Open Systems Interconnection）七层参考模型**是国际标准化组织（ISO）制定的网络分层标准。

|层级|名称|英文|主要功能|数据单位|典型设备/协议|
|---|---|---|---|---|---|
|**第7层**|应用层|Application|为应用程序提供网络服务接口|报文（Message）|HTTP、FTP、SMTP、DNS|
|**第6层**|表示层|Presentation|数据格式转换、加密解密、压缩解压|报文（Message）|SSL/TLS、JPEG、ASCII|
|**第5层**|会话层|Session|建立、管理、终止会话连接|报文（Message）|RPC、NetBIOS|
|**第4层**|传输层|Transport|进程到进程的通信、可靠/不可靠传输|报文段（Segment）/ 用户数据报（Datagram）|TCP、UDP|
|**第3层**|网络层|Network|主机到主机的通信、路由选择|分组/包（Packet）|IP协议、路由器|
|**第2层**|数据链路层|Data Link|相邻节点间的传输、差错检测|帧（Frame）|以太网、交换机、网卡|
|**第1层**|物理层|Physical|传输比特流、电气信号|比特（Bit）|网线、光纤、集线器|

> 📝 **助记口诀**：应表会传网链物（从上到下）
> 或者：物链网传会表应（从下到上）
> 
> 

#### 各层详细解读

**第1层：物理层**

- 在物理媒体上传输和接收电磁信号

- 把链路层的0101比特流转换成电信号/光信号/无线电波

- 关心的问题：用什么电压表示0和1？一根线有几根芯？接口长什么样？

**第2层：数据链路层（简称链路层）**

- 负责**相邻两个节点**之间的数据传输

- 以**帧（Frame）**为单位

- 功能：封装成帧、差错检测、MAC寻址、流量控制

- 典型设备：**交换机**、网卡

**第3层：网络层**

- 负责**源主机到目标主机**的端到端通信

- 解决"比较远"的通信问题（跨多个网络）

- 核心功能：**路由选择**、分组转发

- 以**分组/包（Packet）**为单位

- 典型设备：**路由器**

- 典型协议：IP协议（IPv4、IPv6）

> 📝 **你的笔记补充**：
> "网络层实现主机原点和主机\.\.\." → 完整说法是：**网络层实现源主机到目的主机之间的端到端通信**。
> 
> 

**第4层：传输层**

- 负责**进程到进程**的通信

- 一台主机上可以同时运行多个网络程序（进程），传输层要区分数据是给哪个进程的

- 通过**端口号**来标识不同的进程

- 两种服务模式：

    - **面向连接**：TCP协议，可靠传输，类似打电话

    - **无连接**：UDP协议，不可靠但速度快，类似发短信

**第5\-7层：上三层**

- 这三层在实际的TCP/IP体系中通常合并为**应用层**

- **应用层**：直接为用户的应用程序提供服务

- 按应用层协议交换应用报文

- 实现上传、下载、浏览、游戏等具体功能

### 4\.4 TCP/IP四层模型（实际使用的模型）

OSI七层是理论模型，实际互联网使用的是**TCP/IP四层模型**：

|OSI七层|TCP/IP四层|说明|
|---|---|---|
|应用层|**应用层**|合并了OSI的5、6、7层|
|表示层|||
|会话层|||
|传输层|**传输层**|与OSI对应|
|网络层|**网际层**|与OSI网络层对应|
|数据链路层|**网络接口层**|合并了OSI的1、2层|
|物理层|||

## 五、交换机 vs 路由器 ⭐⭐⭐重点

这是你笔记中提到不太明白的部分，这里详细讲解。

### 5\.1 交换机（Switch）

**工作层次**：数据链路层（第2层）

**核心功能**：在同一个局域网内，根据**MAC地址**转发数据帧。

#### 交换机是怎么工作的？

1. 交换机有很多端口，每个端口连一台电脑

2. 交换机会维护一张**MAC地址表**，记录"哪个端口连了哪个MAC地址"

3. 当一个数据帧到达交换机时：

    - 交换机读取帧中的**目的MAC地址**

    - 查MAC地址表，找到对应的端口

    - 把帧从那个端口转发出去

    - 如果找不到，就**广播**（从所有其他端口发出去）

> 📝 **类比**：交换机就像公司内部的前台，只负责在公司内部传递信件。它认识公司里每个人的工位（MAC地址），收到信后直接送到对应工位。
> 
> 

#### 交换机的特点

- 工作在数据链路层，识别MAC地址

- 用于**局域网内部**的设备互联

- 转发速度快（硬件转发）

- **不能跨网段**，不能连接不同的网络

### 5\.2 路由器（Router）

**工作层次**：网络层（第3层）

**核心功能**：在不同网络之间，根据**IP地址**转发数据包，选择最佳路径。

#### 路由器是怎么工作的？

1. 路由器维护一张**路由表**，记录"去哪个网络应该走哪个端口/下一跳"

2. 当一个数据包到达路由器时：

    - 路由器读取IP包头中的**目的IP地址**

    - **查询路由表**，找到最佳的下一跳地址和出端口

    - 把数据包从对应端口转发出去

    - 同时把TTL（生存时间）减1

> 📝 **你的笔记补充**：
> "端口到网络什么查询路由表，然后再出去" → 完整说法是：
> **路由器收到数据包后，根据目的IP地址查询路由表，找到对应的出端口和下一跳，然后将数据包从该端口转发出去。**
> 
> 

> 📝 **类比**：路由器就像城市之间的快递中转站。它不认识具体某个人，但它知道"去北京的包裹走京广线，去上海的走京沪线"。每个包裹上写着目的地城市（IP地址），路由器根据目的地选择路线。
> 
> 

#### 路由器的特点

- 工作在网络层，识别IP地址

- 用于**不同网络之间**的互联

- 可以选择最佳路径（路由算法）

- 转发速度相对慢一些（软件/硬件转发）

- 可以隔离广播域

### 5\.3 交换机 vs 路由器对比表

|对比项|交换机（Switch）|路由器（Router）|
|---|---|---|
|工作层次|数据链路层（第2层）|网络层（第3层）|
|识别地址|MAC地址|IP地址|
|核心表|MAC地址表|路由表|
|适用范围|同一个局域网内|跨网络、跨网段|
|转发速度|快（硬件交换）|相对慢|
|广播处理|转发广播（同一广播域）|隔离广播（不转发广播包）|
|主要功能|帧转发、局域网互联|路由选择、跨网转发|
|典型应用|家庭/企业局域网|互联网骨干、家庭网关|

### 5\.4 举个例子：你在家刷B站的过程

```
你的手机 → 家里的WiFi路由器（交换机功能） → 光猫 → 
运营商的交换机 → 运营商的路由器 → 互联网骨干路由器 → 
B站所在机房的路由器 → B站的交换机 → B站服务器
```

在这个过程中：

- **交换机**负责在每个局部网络里把数据送到正确的设备

- **路由器**负责在不同网络之间选择路线，把数据从你家送到B站机房

## 六、两种交换方式

### 6\.1 电路交换（Circuit Switching）

**原理**：通信前先建立一条专用的物理通路，通信过程中整条通路被双方独占，通信结束后释放。

**三个阶段**：

1. 建立连接（呼叫）

2. 数据传输

3. 释放连接

**典型例子**：传统的固定电话网

**优点**：

- 实时性好，传输时延小

- 通信质量稳定

**缺点**：

- 线路利用率低（即使没数据传，线路也被占着）

- 建立连接需要时间

### 6\.2 分组交换（Packet Switching）

**原理**：把数据分成一个个小的**分组（包）**，每个分组独立传输，经过多个路由器存储转发，最终到达目的地。

**特点**：

- 不需要预先建立连接

- 每个分组独立选路

- 采用**存储\-转发**机制

- 多个用户可以共享同一条链路（统计多路复用）

**典型例子**：互联网

**优点**：

- 线路利用率高

- 容错性好（一条路断了可以走另一条）

- 适合突发数据传输

**缺点**：

- 有排队时延（路由器缓存满了可能丢包）

- 存在**丢包**问题（网络拥塞时）

- 每个分组都要带地址信息，有额外开销

> 📝 **关于丢包**：
> 分组交换中，路由器有一个缓存队列。如果数据包到达的速度超过了路由器转发的速度，队列就会越来越长，最后队列满了，新来的数据包就会被丢弃——这就是**丢包**。
> 
> 这就是为什么网络拥塞时你会感觉卡顿——数据包丢了，需要重传。
> 
> 

### 6\.3 对比总结

|对比项|电路交换|分组交换|
|---|---|---|
|连接方式|面向连接|无连接（或虚电路）|
|资源占用|独占|共享|
|传输时延|小且稳定|大且可变|
|线路利用率|低|高|
|丢包|不会丢包|可能丢包|
|适用场景|实时语音/视频|数据通信、互联网|

## 七、协议与标准

### 7\.1 什么是协议？

**协议（Protocol）**是对等层的实体在通信过程中必须遵守的规则的集合。

更正式的定义：

> 协议定义了两个或多个通信实体之间交换的**报文格式和次序**，以及在报文传输和/或接收其他事件所采取的**动作**。
> 
> 

### 7\.2 协议的三要素

|要素|说明|类比|
|---|---|---|
|**语法**|报文的格式、数据结构|写信的格式：开头顶格写称呼|
|**语义**|每个字段的含义、要表达的意思|信里写的内容是什么意思|
|**时序/同步**|动作的先后顺序、速率匹配|什么时候发信、多久收到回信|

### 7\.3 PDU：协议数据单元

**PDU（Protocol Data Unit）**是协议数据单元，指的是每一层传输的数据单位。

|层级|PDU名称|说明|
|---|---|---|
|应用层|报文（Message）|应用层数据|
|传输层|报文段（Segment）/ 用户数据报（Datagram）|TCP叫段，UDP叫数据报|
|网络层|分组/包（Packet）|IP数据包|
|链路层|帧（Frame）|数据帧|
|物理层|比特（Bit）|01比特流|

> **封装与解封装**：
> 
> - 发送方：数据从上到下，每一层都加上自己的首部（Header），这个过程叫**封装**
> 
> - 接收方：数据从下到上，每一层都去掉自己的首部，这个过程叫**解封装**
> 
> 

### 7\.4 协议的例子：浏览器访问网页

以浏览器访问服务器为例，看各层协议如何配合：

```
应用层：浏览器按照HTTP协议，构造一个HTTP请求报文
  ↓
传输层：加上TCP首部，包含源端口和目的端口（80）
  ↓
网络层：加上IP首部，包含源IP和目的IP
  ↓
链路层：加上帧首部，包含源MAC和目的MAC
  ↓
物理层：转换成比特流，通过网线发出去
```

到达服务器后，再从下往上逐层解封装，最后应用层拿到HTTP请求报文进行处理。

### 7\.5 标准化组织

|组织|全称|说明|
|---|---|---|
|**IETF**|Internet Engineering Task Force|互联网工程任务组，负责制定互联网标准|
|**ISO**|International Organization for Standardization|国际标准化组织，制定了OSI七层模型|
|**IEEE**|Institute of Electrical and Electronics Engineers|电气和电子工程师协会，制定了802系列标准（如以太网、WiFi）|
|**ITU\-T**|International Telecommunication Union|国际电信联盟，制定通信标准|

### 7\.6 RFC文档

**RFC（Request For Comments）**是互联网标准的文档形式。

- IETF发布的所有标准都以RFC文档的形式存在

- 任何人都可以提交RFC草案，经过讨论和审核后成为正式标准

- RFC编号是唯一的，一旦发布就不会修改，更新会发布新的RFC

- 可以在 [rfc\-editor\.org](https://www.rfc-editor.org/) 查看所有RFC

> 例如：
> 
> - RFC 791：IPv4协议
> 
> - RFC 793：TCP协议
> 
> - RFC 2616：HTTP/1\.1协议
> 
>

## 八、API与操作系统的作用

### 8\.1 什么是API？

**API（Application Programming Interface）**即**应用程序接口**。

简单来说：API是操作系统或软件库提供给应用程序的"调用接口"，应用程序通过API来使用底层提供的功能，而不需要关心底层是怎么实现的。

### 8\.2 网络中的API：Socket编程

在网络编程中，操作系统提供了**Socket（套接字）API**，应用程序通过Socket来进行网络通信。

> 📝 **类比**：
> 你想给朋友寄一封信，你不需要自己造飞机、修公路、建邮局。你只需要把信写好，贴上邮票，扔进邮筒——这个"邮筒"就是API。你通过邮筒这个接口，使用了邮政系统的服务。
> 
> 

### 8\.3 为什么需要操作系统？应用怎么用操作系统？

**操作系统的作用**：

1. **管理硬件资源**：CPU、内存、网卡等硬件都由操作系统统一管理

2. **提供抽象接口**：把复杂的硬件操作封装成简单的API给应用程序用

3. **多任务调度**：让多个程序同时运行，合理分配CPU时间

4. **提供网络协议栈**：TCP/IP协议栈通常实现在操作系统里

**应用程序怎么使用操作系统的网络功能**：

以打游戏为例（你笔记中的例子）：

```
1. 你在游戏里按了一下"攻击"键
2. 游戏程序（应用层）把这个动作描述成一条消息："玩家A发动攻击"
3. 游戏程序调用操作系统提供的Socket API，把消息发给操作系统
4. 操作系统的传输层（TCP/UDP）给消息加上传输层首部
5. 操作系统的网络层（IP）加上IP首部
6. 操作系统的链路层加上帧首部
7. 网卡把数据转换成电信号发出去
8. ...经过网络...
9. 游戏服务器的网卡收到数据
10. 操作系统从下往上逐层解析
11. 操作系统把数据通过Socket API交给游戏服务器程序
12. 游戏服务器处理这个"攻击"消息
```

### 8\.4 基础设施提供的两种服务

网络基础设施（传输层及以下）向上层应用提供两种服务模式：

|服务类型|协议|特点|适用场景|
|---|---|---|---|
|**面向连接服务**|TCP|可靠、有序、有流量控制、有拥塞控制|网页浏览、文件传输、邮件|
|**无连接服务**|UDP|不可靠、无连接、速度快、开销小|视频通话、在线游戏、直播|

## 九、补充知识点

### 9\.1 网卡是什么？

**网卡（Network Interface Card，NIC）**，也叫网络适配器，是计算机与网络之间的硬件接口。

**网卡的作用**：

1. 实现数据链路层的功能（封装成帧、MAC寻址等）

2. 实现物理层的功能（把数据转换成电信号/光信号）

3. 每块网卡都有一个全球唯一的**MAC地址**

> 现在的电脑通常把网卡集成在主板上，不需要单独插卡了，但我们还是叫它"网卡"。
> 
> 

### 9\.2 常见的网络类型

按覆盖范围分：

|类型|英文缩写|覆盖范围|例子|
|---|---|---|---|
|个域网|PAN|几米|蓝牙、USB|
|局域网|LAN|几公里内|家庭网、校园网、企业网|
|城域网|MAN|一个城市|城市有线电视网|
|广域网|WAN|跨城市/国家|互联网骨干|

### 9\.3 什么是互联网？

**互联网（Internet）**是由许许多多的网络通过网络互联设备（路由器）连在一起形成的"网络的网络"。

- 包括科研网、电信网、教育网、企业网等等

- 通过一系列网络交换节点互联

- 遵循统一的TCP/IP协议族

> **注意区分**：
> 
> - **互联网（Internet）**：特指全球最大的那个计算机网络，首字母大写
> 
> - **互连网（internet）**：泛指由多个网络互联而成的网络，首字母小写
> 
> 

### 9\.4 终端设备的种类

网络边缘的终端设备种类越来越多：

- 传统设备：电脑、服务器、手机

- 智能设备：智能电视、智能手表、智能音箱

- IoT设备：摄像头、传感器、智能家居、智能门锁

- 可穿戴设备：VR/AR头盔、健康监测设备

- 工业设备：工业机器人、工控设备

## 十、第一课核心要点总结

1. **网络是什么**：节点 \+ 链路 \+ 协议，实现数据通信和资源共享

2. **三大组成**：网络边缘（主机）、网络接入（接入网）、网络核心（路由器交换机）

3. **分层思想**：每一层负责特定功能，向上提供服务，对等层通信

4. **OSI七层**：应表会传网链物，重点掌握数据链路层、网络层、传输层、应用层

5. **交换机**：第2层设备，根据MAC地址转发，用于局域网内部

6. **路由器**：第3层设备，根据IP地址查路由表转发，用于跨网络互联

7. **两种交换**：电路交换（打电话）vs 分组交换（互联网），分组交换会丢包

8. **协议三要素**：语法、语义、时序

9. **API**：应用程序通过API使用操作系统的网络功能

10. **TCP vs UDP**：面向连接可靠 vs 无连接快速
